1.介绍

本API手册是对于现在使用的由Hexo框架搭建并在GitHub上进行托管的学习平台网站如何进行文章上传，代码如何上传进行记录。本手册适用于所有由Hexo框架搭建的网站。

目前学习平台的网址是https://thistech-top.github.io/，托管在GitHub账号为：dev@thistech.top；密码为：thistechno1。该账户中目前存放着blog的web页面；blog的源代码；还有供他人下载的程序参考示例代码。

2.如何写文章

首先声明使用GitHub搭建的网站在写文章这一点上可能对非程序员有点困难所以我尽量写的傻瓜式一点，下面正式开始。

假如你是第一次使用或是使用一台新的电脑设备，你要想写文章首先第一步就是下载网站的源代码并进行相应的环境配置你需要首先确认你所使用的电脑拥有以下的环境并确保版本较新：

  • Git
  • Nodejs
  • Npm

你可以在cmd中输入命令进行查看，只要在输入命令后出现版本号便OK。

在确保环境没有问题后便可以去github上clone网站源代码到本地上并在dev@thistech.top的ThisTech-top.git与blog_code项目中添加自己的GitHub账户为协作开发者

Step1：进入GitHub对应页面点击setting

Step2：点击Collaborators并搜索自己的GitHub账户

添加完后github会发送一封邮件到你的邮箱中让你确认，在添加完协助开发者后你才可以提交。在下完代码后，你需要在文件的根目录下打开命令行输入npm i，接下来npm便会自动帮你安装完剩下的其他环境。

环境配置完成了，接下来就可以开始写文章，写文章你可以不需要使用任何IDE，一个文本工具和命令行工具。

首先你需要新建文章，在根目录下打开命令行输入hexo new &lt;文章题目&gt;便创建了一个新的文章文件，文本工具打开便可以开始编写文章，这种文件不能使用wps或office打开。

命令行新建文章

使用文本编译工具打开新建的文章

由于文章要上传到网站上，普通的word文本格式是不可以的，但是由于markdown文本格式下编写文章比较复杂，所有我们需要将我们已经写好的word文档转化为markdown文本格式，我常使用的转化工具是https://word2md.com/，他可以一键将word文档转为markdown格式，word文档中的图片会被转化为svg代码，在网站上可以正常显示。要注意的一点是当word文档中的图片过多的时候，最好将文章分段上传。

一键转化markdown格式

在左侧的markdown下方的是Markdown代码我们需要的就是这些东西，右侧markdown代码在网页中的显示效果（通常因为网站样式的不同会有较大的区别，这里的仅供参考）。将markdown代码粘贴到你的文章中，然后在命令行中输入hexo g进行网站重编译

然后继续在命令行中输入hexo d将代码上传并部署到网上。
